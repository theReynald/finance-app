FinanceFlow Documentation

A modern personal budgeting app with financial literacy features — built with React, Vite, Tailwind CSS, and Chart.js.

FinanceFlow is a client-side single-page application that helps users track income and expenses, visualize their spending with interactive charts, and build financial literacy through a curated "Tip of the Day" feature. All data is stored locally in the browser — no backend, no accounts, no servers required.

📖 How to Use This Document
This documentation serves two purposes: (1) A step-by-step build guide — follow Section "Build from Scratch" to recreate the entire app by hand. (2) A reference guide — use the Table of Contents to jump to any component, function, or configuration detail.

🚫 No Backend Required
This is a purely front-end application. There is no server, no database, no authentication. All data lives in the browser's localStorage. This makes it trivial to deploy as a static site on GitHub Pages, Netlify, Vercel, or any web server.

Architecture Overview

The app follows a standard React component tree with state lifted to the root (App.jsx). Data flows down via props; user actions flow up via callback props.

┌──────────────┐ │ App.jsx │ ← Global state (transactions, darkMode) │ │ ← useLocalStorage for persistence └──────┬───────┘ │ ┌────────────┬───────┼────────────┬──────────────┐ ▼ ▼ ▼ ▼ ▼ ┌──────────┐ ┌──────────┐ ┌───────────┐ ┌──────────┐ ┌───────────┐ │ Header │ │TipOfDay │ │Transaction│ │Transaction│ │ Charts │ │ │ │ │ │ Form │ │ List │ │ │ │• totals │ │• self- │ │• onAdd │ │• data │ │• data │ │• toggle │ │ managed │ │ callback │ │• onDelete │ │• darkMode │ └──────────┘ └──────────┘ └───────────┘ └──────────┘ └───────────┘

Data Flow

  1. User adds a transaction → TransactionForm calls onAdd(transaction) → App.jsx appends to transactions state → useLocalStorage writes to localStorage.
  2. User deletes a transaction → TransactionList calls onDelete(id) → App.jsx filters out the transaction.
  3. Charts update reactively — whenever transactions state changes, React re-renders Charts with fresh computed data.
  4. Tip of the Day manages its own state internally via localStorage (independent of the transaction data).

Installation

Clone the repository and install dependencies:

git clone https://github.com/theReynald/finance-app.git
cd finance-app
npm install

This installs all production and development dependencies listed in package.json.

Running the Dev Server

npm run dev

Vite starts a local development server (typically at http://localhost:5173). The page auto-refreshes on any file change thanks to Vite's Hot Module Replacement (HMR).

Production Build

npm run build

Outputs optimized static files to the dist/ folder. To preview the production build locally:

npm run preview

TipOfTheDay Component

FEATURE File: src/components/TipOfTheDay.jsx

Purpose

Displays a single financial literacy tip from a curated list of 54. Shows a new tip each calendar day, with a "New Tip" button to cycle manually. Entirely self-contained — manages its own state.

Props

None. This component is self-managed.

Tip Selection Logic

// On mount: check localStorage for today's saved tip
const saved = localStorage.getItem('financeflow_tip');
if (saved) {
  const { index, date } = JSON.parse(saved);
  if (date === today) return index;       // Reuse today's tip
}
// Otherwise, compute a deterministic daily index
const dailyIndex = Math.floor(Date.now() / 86400000) % financialTips.length;
  • Daily rotation: Uses the current day number (milliseconds / 86400000) modulo the array length to pick a tip. This ensures a new tip each day without server communication.
  • "New Tip" button: Picks a random index (excluding the current one to avoid repeats) and saves it to localStorage.
  • Persistence: The current tip index and date are stored in localStorage under financeflow_tip. If the user refreshes, they see the same tip. On a new calendar day, a new tip is automatically selected.

Visual Design

Warm amber/orange gradient background with a lightbulb icon. Each tip has a title, category badge, and the tip text. The card stands out visually from the blue/gray palette of the rest of the app.

Transaction Data Model

Each transaction is a plain JavaScript object with the following shape:

{
  id: "m1abc2def3",            // Unique ID (timestamp + random string)
  type: "income" | "expense",  // Transaction direction
  amount: 1500.00,             // Positive number (dollars)
  category: "Salary",          // From predefined category list
  description: "March pay",    // User-entered text (optional, max 100 chars)
  date: "2026-03-25"           // ISO date string (YYYY-MM-DD)
}

ID Generation

export function generateId() {
  return Date.now().toString(36) + Math.random().toString(36).substring(2);
}

Combines a timestamp (base-36 encoded) with a random suffix. This produces IDs like m1abc2def3ghij that are unique enough for client-side use without a UUID library.

useLocalStorage Hook

File: src/hooks/useLocalStorage.js

Signature

function useLocalStorage(key: string, initialValue: any): [value, setValue]

How It Works

  1. Initialization: On first render, checks localStorage for an existing value under key. If found, parses the JSON and uses it. Otherwise, falls back to initialValue.
  2. Write-through: A useEffect runs whenever the value changes, serializing it to JSON and writing it to localStorage.
  3. Error resilience: Both read and write operations are wrapped in try/catch. If localStorage is unavailable (private browsing, storage full), the hook degrades gracefully to in-memory state.

Full Source

import { useState, useEffect } from 'react';

export function useLocalStorage(key, initialValue) {
  const [storedValue, setStoredValue] = useState(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  useEffect(() => {
    try {
      window.localStorage.setItem(key, JSON.stringify(storedValue));
    } catch {
      // Silently fail if localStorage is full or unavailable
    }
  }, [key, storedValue]);

  return [storedValue, setStoredValue];
}

Dark Mode Implementation

How It Works (3 layers)

  1. State: App.jsx stores darkMode as a boolean in useLocalStorage('financeflow_darkmode', false).
  2. CSS class: The outermost <div> in App gets className={darkMode ? 'dark' : ''}. When 'dark' is present, all dark: utilities activate.
  3. Tailwind variant: @custom-variant dark (&:where(.dark, .dark *)) in index.css tells Tailwind to match the .dark class on any ancestor (not the OS prefers-color-scheme media query).

Usage in Components

<!-- Light background, dark background when .dark is present -->
<div class="bg-surface dark:bg-surface-dark">

<!-- Light text, dark text when .dark is present -->
<p class="text-text dark:text-text-dark">

cutout: '65%',  // Try '50%' for a thicker ring, or '0%' for a pie chart

Troubleshooting

Dark mode doesn't work

Cause: Missing @custom-variant dark directive in index.css.

Fix: Ensure src/index.css contains this line after the Tailwind import:

@custom-variant dark (&:where(.dark, .dark *));

Without this, Tailwind v4 defaults to prefers-color-scheme: dark (OS-level) instead of the .dark class.

Charts don't render

Cause: Chart.js modules not registered.

Fix: Ensure Charts.jsx includes:

ChartJS.register(CategoryScale, LinearScale, BarElement, ArcElement, Tooltip, Legend);

Data lost after clearing browser cache

Expected behavior. All data is stored in localStorage. Clearing browser data removes it. There is no server-side backup. Consider adding a CSV export feature for data portability.

Port already in use

Cause: Another process is using port 5173.

Fix: Vite automatically picks the next available port. Check the terminal output for the actual URL (e.g., http://localhost:5174).

"Module not found" error on npm run dev

Cause: Dependencies not installed.

Fix: Run npm install to install all packages from package.json.
